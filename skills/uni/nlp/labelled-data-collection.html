Corpus and Dataset

Comparison
  • A corpus is a type of dataset specifically organized for language-related research and often structured and annotated to facilitate linguistic analysis
  • A dataset is a broader term that can refer to any collection of data used for analysis or machine learning, and it can contain various types of data, not just text
  • Corpus linguistics
    • To use corpora to find useful patterns
    • Approaches
      • Use a corpus to confirm or negate a hypothesis
      • Use a corpus to generate a hypothesis
  • Corpus properties
    • Requirements
      • Electronically stored
      • Collection should be quantitative (large)
      • Should contain basic language examples
    • Language
      • Mono-lingual, bilingual, multi-lingual
    • Contents type
      • Literature, newspaper, etc.

Corpus Annotation
...
  • Why is it important?
    • Makes it easier to extract information
      • E.g., language as a lego box
        • Sort the box to tag and organize pieces so you can easily get what you need
    • Reusability
      • Reuse annotated corpus to save time and effort
    • Multi-functionality
      • Can do different kinds of analysis with a well-annotated corpus
        • E.g., grammar and meaning (syntax and semantics)
  • Standards
    • Recoverable
      • Can remove the annotations to get the original text back
    • Extricable
      • The annotations are stored separately from the text
    • Documentation
      • A manual for how you did the annotation so someone else can understand
The MATTER Annotation Cycle
...
  • Model
    • Build a prototype for how to annotate the data and decide on
      • What features are important
      • How will these features be labeled
  • Annotate
    • Use a predefined annotation scheme to annotate the text
      • Tells what to mark and the type of the text
    • Guidelines
      • Determine the goal of the project
      • Determine the set of tags to use
      • Determine the parts of text to be annotated
    • Level of annotation
      • Word/sub-word (e.g., POS tagging)
      • Phrase (e.g., named entity)
      • Document (e.g., document classification)

Inter Annotator Agreements (IAA)
...
Scores that tell much two or more annotators agree with each other
  • Inter-annotator consistency
    • When two annotators give the same label to the sam sentence
  • Intra-annotator consistency
    • When an annotator gives the same label to the same or similar sentences every time
  • High IAA scores
    • Suggest that the annotation guidelines are clear and reliable
  • Problem
    • Simple IAA measure could contain guesses and be misleading
    • Solved using Cohen's Kappa and Fleiss Kappa
⭐ Cohen's Kappa
...
A statistical measure that takes into account the chance of annotators agreeing randomly
  • For comparing two annotators
  • Formula
    • is the percentage of times both annotators agree on the same labels
    • is the percentage of times they agree by random chance
    • ranges between -1 and 1
      • 1 means perfect agreement
      • 0 means no agreement`
      • < 0 means poor agreement

Crowdsourcing
...

Explanation

In crowdsourcing, a company or organization puts out a call on the internet for help with a job. This job can be anything from designing a logo to translating a paragraph from one language to another. People from all over the web – the crowd – can jump in to help out.

  • Ingredients for this to work
    • An Organization
    • A Crowd
    • An Online Environment
    • Mutual Benefit
  • Types of crowdsourcing platforms
    • Paid-for marketplace
      • E.g., Amazon Mechanical Turk
    • Games with a purpose (GWAP)
      • E.g., a game that has players describe what's happening in various images
        • Purpose is to gather descriptions of images to help with training
    • Volunteer-based platform
  • Pros and cons
    • Cheaper than hiring full-time people
    • Can get help from lots of people all over the world
    • Quality control issues
    • Helpers might be unfairly paid
⭐ Workflow
...
  1. Project definition
    • Select the NLP problem
    • Break the problem into smaller pieces
    • Design the task and reward scheme for the crowd
  2. Data and UI preparation
    • Collect and pre-process corpus
    • Set up the tools that people will use to do the work
      • E.g., annotator interfaces
    • Run pilot studies (small tests) to make sure everything works
  3. Running the project
    • Find people that want to help and train them
    • Manage and monitor the tasks
  4. Corpus delivery
    • Evaluate the annotations
