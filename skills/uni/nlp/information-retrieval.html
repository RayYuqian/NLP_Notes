Basics

Information Retrieval (IR) is finding documents of an unstructured nature that satisfies an information need from within large collections

  • Applications

    • Search tasks
    • Find something in a knowledge base

  • Query

    • Textual representation of the information need

  • Information Need

    • Need of a person to require information for solving a problem

  • Relevance

    • How useful the information is to your specific problem
    • Not always clear-cut
      • What's relevant to you might not be to someone else
        • E.g., when you ask about pizza places, you might be interested in the history of pizza, while your friend might just want to know which place has good pizza
    • No definitive answer to what is relevant

  • Relevance feedback

    • User feedback to the system about the relevance

  • Information retrieval cycle

  • Term-document incidence matrix

    • A table that shows if a term appears in a document
    • To answer a query
      • Take the term vectors with bitwise AND
    • Scaling issue
      • Large document collection leads to sparse matrix (too many 0s)
      • Solution -> only record 1s with inverted index
    • Example
      • First column represents terms
      • First row represents documents
      • Query Brutus AND Caesar BUT NOT Calpurnia
        • 110100 AND 110111 and 101111
          • Term vector for Calpurnia is complemented

  • The classic search model

    • Task
      • The problem you want to solve
        • E.g., get rid of mice
    • Info need
      • The info you need to solve the task in a specific way
        • E.g., get rid of mice without hurting them
    • Verbal Form
      • Put info need into words
        • E.g., how do I trap mice alive?
    • Query
      • Words you type in a search engine
        • E.g., mouse trap

      • How a search engine processes a query with multiple words
      • E.g., query Brutus and Caesar

        • Intersect them to find the documents that contain both terms
  • Index creation steps
  • MapReduce
    • A method to create a massive search engine index
    • Master
      • The main control unit that assigns specific tasks to various computers
    • Map phase
      • Split the documents into smaller chunks
      • Process each chunk separately to create a partial index
    • Reduce phase
      • Combine the partial indexes into a final index

Retrieval
...
  • Boolean model
    • An old-school way of searching
    • Use words like AND, OR, and NOT to find documents
  • Vector space model (VSM)
    • Both queries and documents are represented in a vector space
    • Vectors
      • Representations of each document or query
    • Weighting scheme
      • Term frequency
        • The frequency of term in a document
      • Document frequency
        • The number of documents with the term
      • Inverse document frequency
        • A way to figure out how unique a term is
          • Higher weight -> more unique
        • is the total number of documents
      • Tf-idf (best known weighting scheme)
        • and combined
        • Weight terms based on how often they appear across multiple documents
          • Higher weight if a term is frequent in a document but rare in others
    • Example
  • Probabilistic models
    • Predict the likelihood that a document is relevant to your query
Evaluation
...
  • Result sets (unordered)
    • The returned documents
    • Precision, recall, and f-measure
      • Not suited for web search scenario -> MAP and NDCG are better
  • Result lists (ordered)
    • Returned documents ordered according to their relevance for the topic
    • P@k means precision at
      • Look up the top k results and see how many are relevant
  • Relevance judgements
    • The precision-recall-tradeoff
      • More documents retrieved usually lead to lower precision and higher recall
    • Mean average precision (MAP)
      • Average over all intermediate values at relevant docs
      • Higher value means better performance of the search engine
      • Example
    • Normalized discounted cumulative gain (NDCG)
      • Used when different results have different levels of relevance
      • Relevance is graded on a scale (usually 0 to 4)
      • DCG
        • The sum of graded relevance scores
          • is the position of the result in the result list
          • is the graded relevance score at position
      • NDCG
        • Normalized scores

Crawling and indexing - Offline
...
  • Web crawlers
    • Algorithm
      • Start with a list of web pages (seed pages) and visit them
      • Follow the links on those pages to find new pages
      • Keep track of all pages it has visited
    • Does not overload any website with too many visits at once
    • Revisit pages sometimes to see if anything has changed
    • Duplicate detection
      • Look for and remove duplicates after crawling but before indexing
      • Easier to find exact duplicates, harder for partial duplicates
  • Indexing
    • Inverted index
Querying and ranking - Online
...
  • Query types
    • Informational
      • When people want to learn about sth.
      • 40% - 65%
    • Navigational
      • When people want to go to a specific website
      • 15% - 25%
    • Transactional
      • When people want to do sth., like shopping
        • 25% - 35%
    • Other
      • Just exploring
  • Ranking
    • PageRank
      • Ranking is based on
        • The number and quality of links to a page (inlinks)
        • The text in the links to that page (anchor text)
      • Web is represented as a directed graph
      • Inlinks
        • A link of another web page that contains a linking pointing to the web page
      • Outlinks
        • A link in the web page that points to another web page
      • Definition
          • is a damping factor, adjusts the importance of inlinks
          • is the total number of pages
          • is the number of outlinks on page
    • Anchor text
      • Combine the text of a document with the anchor text gives a more accurate result
        • Because anchor texts can provide additional context
      • Google bomb
        • A search with bad results due to maliciously manipulated anchor text

Conceptual indexing for semantic search
...
  • The goal is to make search engine understand user better
  • Handle different types of words
    • Polysemy
      • Have multiple meanings
    • Synonymy
    • Hyponymy
      • Are more specific versions of a general term
  • Approaches
    • Ontology-based (semantic web)
      • Contents are labeled with formal descriptors
      • Queries are translated into formal descriptors
      • Difficult to maintain as someone has to manually label the contents
    • NLP-based
      • Documents analyzed with syntactic and semantic processing steps
      • Index contains more info like connections, concepts, etc.
      • Slow and can take up a lot of computer space
ML for Building Query Models
...
  • Ranking function
    • Score each result to decide the order they should appear in
      • is the user query
      • is the document
      • represents the th feature
        • E.g., term frequency (TF)
      • is a weight assigned to the corresponding feature
      • is an feature function
        • E.g., one feature function might look at term frequency, while another might consider whether the search terms are in the title of the document
    • Adapt logistic regression for ranking
      • Used to predict how likely you'll find a relevant document
        • This will squash the score between 0 and 1
          • Higher value means more relevant
  • Query modeling
    • The process of tweaking the query to improve search results
      • Assign weights to terms in the query
    • Query reduction
      • Remove less important terms
      • E.g., Q = {give information on steps to manage control or protect squirrels}
        • g(Q) = {steps control protect squirrels}
    • Query weighting
      • Decide which terms are more important
      • E.g., Q = {civil war battle reenactments}
        • g(Q) = {0.3 war, 0.2 battle, 0.1 civil, 0.4 reenactments}
    • Query expansion
      • Add more words that can help improve the query
Advertising Approaches
...
  • Database approach
    • Ads are stored in a database
      • Each ad has an attribute called bid phrase (BP)
    • Two strategies for selecting ads
      • Exact match
        • If the query matches BP
      • Advanced match
        • The system expands the search terms in various ways
  • IR approach
    • Ads are documents in an ad index
    • No strict division between exact and advanced matches
