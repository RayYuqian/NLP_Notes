Approaches
...

  • Using hit counts returned by a commercial search engine
    • Estimates might be unreliable
      • Since the information may change over time
    • Only very simple queries are possible
      • No regular expressions, POS tagging, etc.
  • Build corpora through search engine queries
    • Issue automated queries and retrieve pages returned, then process the pages
    • Dependence on the availability of search APIs
    • Bootstrapping
      • Use the gathered information to help gather more information
  • Linguistic crawls of the Web
    • Search the web using a bunch of seed URLs and build its own collection through them
    • Not dependent on commercial search engine
    • Difficult to implement
    • Steps
      1. Visit each URL and saves a copy of the document
      2. Detect outgoing hyperlinks and add them to the list of URLs to be visited next

Data Cleanup
...

  • Web Spam Filtering
    • Web spams are artificially constructed pages
    • Examples
      • Splogs (spam blogs)
      • Automatically generated pages
  • HTML File Pre-Preocessing
    • Remove boilerplate
      • Automatically generated text blocks, e.g., navigation bars, ads
    • Methods
      • Use HTML structure or tags
      • Use language models
    • Preprocessing steps
      1. Frames detection
        • Identify different sections on a page, e.g., title, image, content, etc.
      2. File encoding detection
        • Figure out what the language is
      3. HTML code cleaning
        • Remove extra HTML code not needed for the actual content
      4. Convert HTML entities
        • Convert special codes to original characters
          • E.g., &lt; -> <
  • Content extraction
    • Steps (DOM-based)
      1. Parse the HTML to obtain a Document Object Model (DOM) tree
      2. Use the filters to remove specific nodes in the tree
        • E.g., images, links, ads, etc.
      3. Output the page in HTML or plain text
  • Duplicate Removal
    • Perfect duplicates
      • Identical web pages at different URLs
        • E.g., FAQ pages
    • Near duplicates
      • Pages with very similar content but different designs
      • Shingling algorithm
        • Could be used to detect plagiarism
        1. Define shingles of document
          • A shingle is like a small snippet of a document
        2. Compute shingle fingerprint of each
          • A unique identifier for a document based on its shingles
        3. Compute super-shingles
          • Bigger chunks created by combining shingles
        4. Compare documents that share a super-shingle
        5. Discard documents that have more than a certain number of super-shingles
    • Partial duplicates
      • Pages with some content copied from other web pages
  • Content cleaning
    • Confusion set disambiguation
      • Picking the right word from a set of similar ones based on context
      • Usage
        • Improve spelling, understand word meanings
      • Steps
        • Replace all occurrences of confusion set members with a marker
        • The system chooses one of the members in the confusion set
      • E.g., {principle, principal}, {then, than}, ...
  • Detect erroneous pages
    • Detect and filter out web pages that have too many errors
    • A filter is contains
      • An error dictionary
      • A filter threshold (max # of errors to tolerate)
    • Filter on the document level
      • E.g., discard documents smaller or larger than a given size
    • Filter on the sentence level
      • E.g., discard sentences that are too short or too long

Research Problems
...

  • Non-word error detection

    • Catch and fix mistakes that result in words which don't exist in the language
    • Methods
      • Analyze letter n-grams
        • Check the n-grams against a huge list to see how often they appear
          • Likely to be a type if it's rare
        • Very useful for OCR tasks
      • Dictionary lookup
        • Look up the word in a digital dictionary

  • Isolated-word error detection

    • Fix spelling mistakes in single words without considering the context
    • Steps
      1. Find spelling mistakes
      2. Make some guesses of the correct forms
      3. Choose the best correct word
    • Minimum edit distance algorithm
      • Find out how similar two words are by counting the smallest number of edits needed to change one word into the other
      • Steps
        1. Set up the table
        2. Loop through each letter of the two words and do one of the actions
          • No change (if matches)
          • Insertion (add a new letter)
          • Deletion (remove a letter)
          • Substitution (change a letter to another)
        3. Add up the cost
          • Cost is 0 if they match, 1 otherwise
      • Example

  • Context-dependent word correction

    • Using context bi-, uni- grams to correct words
      • Each confusion word has counts of a bigram (its left and right word) and two unigrams (left and right words)
    • Types of classifiers
      • Choose the confusion word with the most counts of context bigrams
      • ... most counts of left unigram
      • ... most counts of right unigram
      • Choose the most frequent confusion word
