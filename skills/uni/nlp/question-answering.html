Introduction
...

  • IR vs. QA/RAG (retrieval augmented generation)
    • Input
      • IR - keyword-based queries
        • E.g., polar bears life span
      • QA/RAG - natural language questions
        • E.g., How long do polar bears live?
    • Output
      • IR - whole documents
      • QA - precise and concise answers
  • Applications
    • Cross-language QA
      • Questions on in one language
      • Answers in several other languages
        • Possible translation errors
    • Opinion QA
      • Questions about what people think or feel
      • No single correct answer, multiple perspectives required
    • QA with the web
      • Use web as the primary/supplementary corpus

Question Processing
...
  • Query formulation
    • Remove stop words
    • Stemming or lemmatisation
    • etc.
  • Query classification
    • Question types
      • Factoid
        • Ask for quantitative information
        • Generally answered with data
        • E.g., How long do polar bears live
      • Definition
        • Seek explanations of what sth./sb. is
        • Answered with explanatory text that provides more context
        • E.g., Who is Britney Spears?
      • Opinion
      • More difficult types
        • How? Why?
    • Answer types
      • Categorized by its meaning and the context of the question
      • E.g., What is the capital of France
        • Semantic category -> city
Passage Retrieval
...
  • Document retrieval
    • Identify N most relevant documents
      • Often performed by a search engine
  • Passage retrieval
    • Retrieve N most relevant passages from the documents
    • Passages
      • Sentences, paragraphs, sections
    • Filtering
      • Remove passages that do not contain potential answers
        • Use expected answer type and the named entities in the passages
    • Ranking based on
      • How many relevant named entities they contain
      • How many keywords from the question they include
      • The longest sequence of question keywords they have
      • ...
Answer Identification
...
  • Find the best answer within the selected passages
  • Methods
    • Pattern extraction
      • Use known patterns (like regular expressions) that fit the expected type of answer
    • Span labeling
      • Identify the exact part (span) of the text that provides the answer
    • Redundancy-based (AskMSR system)
      • Use the web data
      • Answer redundancy
        • The more texts that contain the answer, the more likely the answer is correct
      • Mine n-grams
        • Look at how often each n-gram appears in the summaries
      • Filter n-grams
        • Remove n-grams that do not match the answer type
      • Tile n-grams
        • Merge the n-grams to form n-best answers
  • Automatic answer pattern learning
    1. Start with a seed example
    2. Use that to search and download the top 1000 documents
    3. Keep only the sentences that contain both the question and answer terms
    4. Create a suffix tree to extract
      • The longest matching substrings that span <NAME> and <ANSWER>
    5. Measure the precision
    • E.g., When was Mozart born?
      • Step 1
        • Mozart, 1756
      • Step 3
        • Mozart was born in 1756
        • ... born in 1756, Mozart ...
      • Step 4
        • <NAME> was born on <BIRTHYEAR>
        • ... born in <BIRTHYEAR>, <NAME> ...
    • Perform better for web data since it contains more candidate answers
    • Does not handle long-distance dependencies

⭐ QA over Linked Data (QALD)
...

  • The classic web
    • Databases contain HTML files
    • HTML files are connected by hyperlinks
    • URLs used to retrieve HTML files
    • Content structure not friendly for computers
    • Cannot ask complex questions
  • Linked data
    • Use semantic web technologies to link data
      • Use special formats that help computers understand data relationships
    • Can publish structured data on the web
    • Can set links between data from different sources
    • Principles
      • Use URIs (uniform resource identifier)
      • Provide useful RDF (resource description framework) info
    • RDF
      • Breaks every piece of info into triples
        • Subject
        • Predicate - tells the relationship between subject and object
        • Object
      • E.g. http://example.org/place/Brussels is the capital of Belgium
        • Subject - http://example.org/place/Brussels
        • Predicate - is the capital of
        • Object - Belgium

Evaluation
...

  • CLEF judgement (answer assessment)

    • Correct
      • The answer is spot on and supported by evidence from the correct source
    • Inexact
      • The answer contains unnecessary info or misses relevant info
    • Unsupported
      • Either got the wrong source or nothing in the source to back up the answer
    • Incorrect
      • The answer doesn't match the question

  • Mean Reciprocal Rank (MRR)

    • Reciprocal rank (RR)
      • The score you get based on when you make the correct guess for a question
        • 1 if the first answer is correct
        • 0 if not correct answers
    • MRR
      • Mean of the RR over all questions

  • Confidence-weighted score (CWS)

    • Show how good the system is at being confident about the right answers

  • Accuracy

    • Average of over all questions
      • 1 if the first answer is relevant, 0 otherwise

LLMs in Conversational Search
...

  • Few-shot prompting
    • Provide the LLM a few examples to learn what to do
  • Reranking
    • Pointwise method
      • Decide if one item is relevant
    • Listwise method
      • Rank the whole list by relevance
    • Pairwise method
      • Compare two items at a time to decide which one is more relevant
  • Reader
    • Passive reader
      • Summarize information given
    • Active reader
      • Trained LLM that uses the IR engine to actively query for missing info
    • WebGPT
      • Can browse the web to answer questions
      • Get better over time by learning which answers are most helpful
