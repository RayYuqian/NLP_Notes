⭐ Approaches
...
  • Dictionary based
    • Relies on having a set of dictionaries, each one specific to a language
    • E.g., Imagine you have a list of words that are unique to the English language
      • If you receive a piece of text and you find a significant number of these English-specific words in it, you can determine that the text is likely in English
  • Language modeling
    • Captures patterns that correspond to a language
    • E.g., given a new french text that a model hasn't seen
      • If it fits the patterns of a French model well, it might determine it's French text
  • Classification methods
    • Involves training a model on labeled text with their corresponding languages
    • E.g., if the text has a lot of "ñ" and "ll," it might be classified as Spanish
Language Modeling
...
  • Word-based modeling
    • Build language models using words from a training corpus
    • Zipf's law
      • Prioritizes words by frequency, using the most common ones
    • Frequency sorting
      • Words are arranged from most to least frequent]
    • Usages
      • Predict the next word in a sequence
      • Determine word frequencies
      • Understand language structure and usage
  • Character N-gram-based modeling
    • Build language models using character n-grams (2 to 5)
      • By counting and analyzing the frequency of these n-grams
    • Usage
      • Address spelling errors in OCR
        • As it doesn't rely on correct spelling to understand and predict text

⭐ Naive Bayes Classifier
...
  • Bayes' theorem
    • Allows us to reverse the conditional probability of events A and B
  • Disjoint events
    • Events that cannot occur at the same time
  • Definition
    • Find the most probable class based with attribute values ()
      • Likelihood is calculated using for all classes
    • Assume the attributes (features) are independent
  • Application
    • Text classification
    • Training is very fast
    • Binary, numeric and nominal features can be mixed
      • Nominal features do not have a particular order and are not quantitative
⭐ Support Vector Machines (SVM)
...
A technique used to categorize data into two groups by drawing a line in between
  • Max margin
    • The maxed distance between the nearest data points on both sides
  • Support vectors
    • Data points closest to the max margin hyperplane
  • *Kernel function
    • A mathematical function used when the data is not linearly separable
    • Transforms the mixed data into a (high-dimensional) space
      • Makes it easier to separate the data
    • RBF (Radial Basis Function)
      • Can handle data that is spread out in all directions
    • Tree kernel
      • Work with tree-like data, e.g., parsing trees
  • Advantages and limitations
    • Good for when there are a lot of features
    • Flexible, because you can choose different kernels
    • Robust, because it can handle noisy data
    • Can be slow and computationally complex with large amounts of data
    • Trial-and-error to find the right kernel
    • Black box

Intrinsic Evaluation
...
Intermediary measures that tell the model's performance in a controlled environment
  • Accuracy
    • Used when output variable is categorical or discrete
  • Precision
    • Used in classification tasks, especially where false positives are more costly
      • E.g., disease predictions
  • Recall
    • Used in classification tasks, especially where retrieving positive results is more important
      • E.g., e-commerce search and other IR tasks
  • F1 score
    • Used with accuracy in most classification tasks
      • E.g., sequence-labeling and retrieval-based QA
Extrinsic Evaluation
...
See how the model's work affects the real-world tasks it was designed to help with

For example, if a model is recommending news articles, an extrinsic measure would be to find out whether people click on the articles more often and spend more time reading them

Named Entity Recognition (NER)
...
  • Identify proper names within text and classify them into predefined categories
    • E.g., person, location, email addresses, etc.
  • Named entity (NE) is a noun phrase
    • E.g., Marriot Hotel (NE) vs. Hotel
  • Crucial for information extraction system
  • Challenges
    • A named entity could have variations
      • E.g., Albert Einstein, Albert, Dr. Albert Einstein
    • Ambiguity
      • E.g., Washington (location vs. person)
    • Grey areas in category definitions caused by metonymy
      • Refer to sth. by the name of sth. else that is closely connected with it
        • E.g., The White House issued a statement.
          • The White House refers to the officials working there
Approaches
...
  • Dictionary loop-up
    • Use dictionaries to identify NEs
    • Simple and precise
    • Costly and time-consuming
  • Rule based
    • Need rules to identify NEs
      • E.g., using capitalization of words to infer that they are proper nouns
    • Flexible and transparent
    • Require extensive grammatical knowledge
  • Machine learning
    • Classification task to tag sequence by learning from labeled training text
Entity Linking
...
Definition

The process that involves identifying pieces of text that refer to entities (like people, places, etc.) and connecting these mentions to a unique identifier within a knowledge base

  • Knowledge base (KB)
    • A structured database of facts which includes information about various entities
    • E.g., Wikipedia
  • Scope
    • Limited to recognizing entities that already exist in the KB
  • Importance
    • A way to link the raw and noisy data on the Web to the KB
  • Steps
    • Mention detection
      • Detect all "linkable" phrases
        1. Build a dictionary of entities with all name variants
        2. Compare all n-grams with the dictionary
        3. Filter out undesired entities
    • Candidate selection
      • Generate a set of candidate entities for each mention
        • Keep only candidates above a rank threshold
    • Candidate disambiguation
      • Select a single entity (or none) for each mention based on context
      • Prior importance
        • How often an entity is referenced in the KB
        • How frequently it's mentioned in the document
      • Contextual similarity
        • Compare the context with the description of the entity in the KB
      • Coherence
        • Linked entities should be topically related

Language Model (LM)

A statistical model over word sequenceses that can be used to generate word sequences and estimate the probability of a sequence.

  • Trigram model
    • Predicts the next word in a sequence based on the two preceding words
  • Why ChatGPT is much better?
    • Embeddings instead of word symbols
      • Words with similar meanings are closer to each other
    • Transformer architectures that are effective at processing sequential data
    • Attention mechanism that allows the model to focus on different parts of the input
  • Abilities and limitations
    • Good at understanding and generating human language
    • Limited only to language processing
    • Can't perform logical reasoning
    • Can increase misinformation on the internet
    • Sources are not tracked
